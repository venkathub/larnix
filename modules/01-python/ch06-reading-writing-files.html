Reading & writing files (CSV/JSON)

Beginner Browser Stable

The hook

Enough hand-typing rows. The real habits.csv — all 35 days — is sitting right next to this page, and now you’ll load it. When you do, you’ll see something important: real data is messy. Some step counts have commas (7,500), some sleep values carry a stray h (8h), some moods are padded (ok), and a few fields are blank. Loading the file is step one; the rest of the module makes it clean.

You’ll need from before: types and string-to-number conversion (Ch1), and dicts (Ch4) — because each CSV row will arrive as a dict.

Reading a CSV

Open a file with open(...), ideally inside a with block so it closes itself:

with open("data/habits.csv") as f:
    text = f.read()

But you rarely want the raw text — you want rows. The csv module’s DictReader reads each row into a dict keyed by the column headers:

import csv

with open("data/habits.csv") as f:
    rows = list(csv.DictReader(f))

rows[0]   # {'date': '2026-01-01', 'steps': '7500', 'sleep_hours': '7.5', 'mood': 'good'}

Two things to notice: each row is exactly the dict shape from Chapter 4 (row["steps"]), and every value is a string — '7500', not 7500 — so you’ll convert with int()/float() before any maths (Chapter 1).

Working with JSON

JSON is a text format for structured data, used everywhere data is sent between programs. The json module converts both ways:

import json

text = json.dumps({"steps": 7500, "mood": "good"})   # object -> JSON text
obj = json.loads(text)                                # JSON text -> object

dumps (“dump string”) serialises; loads (“load string”) parses it back.

Writing a file

csv.DictWriter writes dict rows back out — useful for saving a cleaned table:

with open("clean.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["date", "steps"])
    writer.writeheader()
    writer.writerows(some_rows)
Note🧱 For Java developers

with open(...) as f: is Python’s try-with-resources — the file closes automatically at the end of the block. DictReader and the json module replace the BufferedReader/parsing and Jackson/Gson boilerplate you’d reach for in Java, in a couple of lines.

⚠️ Watch out: in the browser, files live in an in-memory filesystem. The habits file was loaded into it for this page, and anything you write lives there too — but it vanishes when you close the tab. That’s perfect for learning, and exactly how the in-browser runner stays ₹0.

Look closely at those rows — steps of '7,500', a blank field, a sleep_hours of '8h'. That mess is real, and cleaning it is the next few chapters.

You read a real file, transformed it, and wrote a new one — entirely in the browser’s in-memory filesystem.

Key Takeaways

Key Takeaways

  1. csv.DictReader reads each row into a dict keyed by the header; every value is a string, so convert before doing maths.
  2. json.dumps turns a Python object into JSON text; json.loads turns it back.
  3. csv.DictWriter writes dict rows back out to a file.
  4. In Pyodide, files live in an in-memory filesystem — great for learning, but writes disappear when the tab closes.

A CSV row arrives as a dict of strings. Complete the marked line so steps_as_int reads the steps field and converts it to a whole number. Replace ____, then press Run.

def steps_as_int(row):

row["steps"] is the string "7500"; wrapping it in int(...) gives the number 7500.

Exercise 2 — Implement (≈10 min) 🟡

Write total_steps(rows) — sum the steps across a list of dict rows, converting each from text to an int. Complete the marked line.

If you’re stuck Inside the loop you want this row’s steps as a number — int(row["steps"]) — added to the running total.
Show solution 
def total_steps(rows):
    total = 0
    for row in rows:
        total += int(row["steps"])
    return total
Each row’s "steps" string is converted to an int and added to total, which the function returns at the end.

Exercise 3 — Stretch (optional, ≈10 min) 🔴

Rubric — a good answer:

Show one possible answer 
import csv

with open("data/habits.csv") as f:
    rows = list(csv.DictReader(f))

missing = sum(1 for r in rows if r["steps"] == "")
print("Rows:", len(rows))

DictReader gives the rows; the count of those whose steps is "" tells you how much patching the cleaning step will need — a real first look at data quality.
