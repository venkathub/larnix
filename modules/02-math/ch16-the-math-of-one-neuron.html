The math of one neuron

Intermediate Browser Stable

The hook

This is the chapter the whole module has been building toward. Every piece you’ve met — a dot product, a derivative, a gradient, the chain rule — clicks together here into the smallest learning machine there is: one neuron. Deep learning stacks this exact unit by the million. Understand one, and the rest is repetition.

A neuron does two small things. It takes a row of numbers and mixes them into a single score (a weighted sum). Then it bends that score into a prediction between 0 and 1 (an activation). That’s the forward pass. Learning is the backward pass: measuring how wrong the prediction was, then using the chain rule to work out how to nudge each weight to be a little less wrong next time.

We’ll do both passes by hand, in code you can read line by line, and then prove our hand-derived gradient is correct by checking it against a numeric estimate.

You’ll need from before: the dot product (Ch5), the derivative as sensitivity (Ch6), the gradient (Ch7), the chain rule (Ch8), and NumPy arrays (Ch15).

The forward pass: from inputs to a prediction

A neuron holds two things it can adjust: a weight for each input (w) and a single bias (b). Given an input vector x, it computes:

  1. The weighted sum z = w · x + b. This is the dot product from Chapter 5 — a similarity score between the weights and the input — shifted by the bias.
  2. The activation a = σ(z), where σ (sigmoid) is 1 / (1 + e^(−z)). It squashes any real number into the range 0 to 1, so the output reads like a probability.

Let’s use one concrete example all the way through. Input x = [2, 3], weights w = [0.5, −1.0], bias b = 0.5, and the true answer we wanted is y = 1.0.

Running it: z = 0.5·2 + (−1.0)·3 + 0.5 = −1.5, the sigmoid squashes that to a ≈ 0.182426, and since we wanted y = 1.0 the squared-error loss is (0.182426 − 1.0)² ≈ 0.668428. The prediction is far too low — so the neuron has something to learn.

The loss: a number that says how wrong we are

To learn, the neuron needs a single number measuring its mistake. We use squared error: L = (a − y)². It’s zero when the prediction is perfect and grows as the prediction drifts from the truth. Squaring (like in Ch11’s variance) keeps it positive and punishes big misses more. Learning means changing w and b to make L smaller — and to know which way to change them, we need the gradient of L.

The backward pass: the chain rule does the work

The loss depends on a, which depends on z, which depends on each weight. That’s a chain — exactly Chapter 8. To get how the loss responds to weight w_i, multiply the links:

\[\frac{\partial L}{\partial w_i} = \frac{\partial L}{\partial a} \cdot \frac{\partial a}{\partial z} \cdot \frac{\partial z}{\partial w_i}\]

Each link is something you already know how to differentiate:

  • dL/da = 2(a − y) — the derivative of the squared error.
  • da/dz = a(1 − a) — the derivative of the sigmoid (a tidy result worth remembering: the sigmoid’s slope is its own output times one minus itself).
  • dz/dw_i = x_i and dz/db = 1 — the weighted sum is linear in each weight, so its slope is the matching input itself (and 1 for the bias).

Multiply the first two once to get dL/dz, then fan it out to every weight by its input. Here’s the whole backward pass:

That gives dL/dz ≈ −0.243877, so grad_w ≈ [−0.487753, −0.731630] (each is dL/dz times its input, 2 and 3) and grad_b ≈ −0.243877. The negative signs say: to reduce the loss, increase these weights and the bias — which would push the prediction up toward the target y = 1.0, exactly what we want.

Gradient checking: proving the hand-derivation is right

How do we know that calculus is correct and not a slip of algebra? We check it numerically. A derivative is a slope, and we can estimate any slope without calculus by nudging the input a tiny amount eps each way and measuring the rise over run: (L(p + eps) − L(p − eps)) / (2·eps). If our analytic gradient matches this numeric estimate to many decimals, it’s right.

The two gradients agree to a maximum error of about 3.27e-11 — eleven decimal places. The hand-derived chain rule is correct. That tiny leftover is rounding in the finite-difference estimate, not a mistake in the calculus.

This is not a toy check: gradient checking is exactly how practitioners catch bugs when they implement backpropagation in a real network. One neuron or one billion, the recipe is the same — forward pass, loss, chain rule backward, verify.

Key Takeaways

Key Takeaways

  1. A neuron’s forward pass is a dot product plus a bias (z = w·x + b) followed by an activation (a = σ(z)) that squashes the score into 0..1.
  2. The loss (a − y)² is one number measuring how wrong the prediction is; learning means making it smaller.
  3. The backward pass uses the chain rule to get the gradient of the loss for every weight: dL/dw_i = 2(a − y) · a(1 − a) · x_i.
  4. A numeric gradient check (finite differences) confirms the hand-derived gradient — here they matched to about 3e-11.
  5. This single unit, stacked and trained, is all of deep learning. You’ve now done its math by hand.

Exercise 1 — Guided (≈4 min) 🟢

The activation is the sigmoid. Complete sigmoid(z) so it returns 1 / (1 + e^(−z)). Replace ____, then press Run.

If you’re stuck e^(−z) is np.exp(-z). The formula is one divided by (1 + np.exp(-z)).
Show solution 
import numpy as np

def sigmoid(z):
    return 1.0 / (1.0 + np.exp(-z))
At z = 0, e^0 = 1, so the value is 1 / (1 + 1) = 0.5 — the sigmoid’s midpoint. Large positive z drives it toward 1; large negative z toward 0.

Exercise 2 — Implement (≈8 min) 🟡

Now the whole forward pass. Write neuron_forward(x, w, b) that returns the activation σ(w · x + b). Use NumPy for the dot product. Complete the marked line.

If you’re stuck The weighted sum is the dot product of w and x plus the bias. With NumPy arrays that’s np.array(w) @ np.array(x) + b.
Show solution 
import numpy as np

def neuron_forward(x, w, b):
    z = np.array(w) @ np.array(x) + b
    return 1.0 / (1.0 + np.exp(-z))
np.array(w) @ np.array(x) is the weighted sum; adding b shifts it; the sigmoid squashes the result. For our example z = −1.5, so the output is ≈ 0.1824.

Exercise 3 — Implement (≈8 min) 🟡

The key step of the backward pass. Write dL_dz(a, y) — the gradient of the squared-error loss with respect to the weighted sum z. By the chain rule this is 2(a − y) · a(1 − a). Complete the marked line.

If you’re stuck Two links of the chain multiplied: dL/da = 2*(a - y) and da/dz = a*(1 - a). The answer is their product, 2 * (a - y) * a * (1 - a).
Show solution 
def dL_dz(a, y):
    return 2 * (a - y) * a * (1 - a)
At a = 0.5, the sigmoid slope a(1 − a) is 0.25. If the truth is y = 1.0 we under-predicted, so the gradient is negative (−0.25) — pushing the prediction up. If y = 0.0 we over-predicted, so it’s positive (+0.25) — pushing it down.

Exercise 4 — Stretch (optional, ≈10 min) 🔴

Open-ended, rubric-graded (not auto-checked). Put it all together: write a forward pass, derive grad_w and grad_b analytically, then gradient-check them against finite differences for an input of your choice. Print the max error.

Rubric — a good answer:

  • Computes the forward pass and the analytic gradient for w and b.
  • Estimates the same gradient numerically with (L(p+eps) − L(p−eps)) / (2·eps).
  • Reports the max absolute difference and confirms it is tiny (well under 1e-6).
Show one possible answer 
import numpy as np

def sigmoid(z):
    return 1.0 / (1.0 + np.exp(-z))

x = np.array([1.0, -2.0, 0.5])
w = np.array([0.3, 0.8, -0.5])
b = -0.1
y = 0.0

def loss_at(w, b):
    return float((sigmoid(w @ x + b) - y) ** 2)

a = sigmoid(w @ x + b)
dL_dz = 2 * (a - y) * a * (1 - a)
analytic = np.append(dL_dz * x, dL_dz)   # grad_w then grad_b

eps = 1e-5
numeric = []
for i in range(len(w)):
    wp = w.copy(); wp[i] += eps
    wm = w.copy(); wm[i] -= eps
    numeric.append((loss_at(wp, b) - loss_at(wm, b)) / (2 * eps))
numeric.append((loss_at(w, b + eps) - loss_at(w, b - eps)) / (2 * eps))

max_error = float(np.max(np.abs(analytic - np.array(numeric))))
print("max error = {:.2e}".format(max_error))
Any reasonable input gives a max error around 1e-10 or smaller — confirmation that the hand-derived gradient is correct, on data different from the chapter’s example.

Where this fits / what’s next

You have now done, by hand, the exact computation that runs billions of times inside a trained network: forward pass, loss, chain-rule backward, gradient check. Stacking these neurons into layers and repeating the nudge is gradient descent — the engine of every model in this course.
