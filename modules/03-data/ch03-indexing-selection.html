Indexing & selection

Beginner Browser Stable

The hook

A table is rarely interesting all at once. The questions you actually ask are small slices of it: what species is in row 0? show me the first five rows. which penguins weigh more than four kilograms? Each of those is a selection — you point at part of the table and pandas hands you back exactly that part.

In the last chapter you loaded the Palmer Penguins table and sized it up. Now you learn to reach into it. There are three moves, and almost everything you do with data is a combination of them:

  • Pick a column by name — df['species'].
  • Pick rows — by their position with .iloc, or by their label with .loc.
  • Filter rows with a boolean mask — keep only the rows that pass a test, like body_mass_g > 4000.

Get these three reflexes and a DataFrame stops being a wall of numbers and becomes something you can interrogate.

You’ll need from before: a DataFrame is a table of named columns, and df['col'] returns one column as a Series (M3 Ch2). We build straight on that.

Picking a column

The simplest selection is one column, by name, in square brackets. What comes back is a Series — the labelled column you met last chapter.

df["species"] is a Series, so every Series tool carries over. A column of text is often best summarised by counting its values:

That tallies Adelie 152, Gentoo 124, Chinstrap 68. To pull several columns at once, pass a list of names inside the brackets — note the double brackets, an outer pair for indexing and an inner pair for the list. The result is a smaller DataFrame, not a Series:

One name in brackets gives a Series; a list of names gives a DataFrame. Keep that distinction in mind — it explains a lot of “why did I get a different shape?” surprises later.

Picking rows by position with .iloc

Columns have names, but rows are easiest to reach by position: the first row, the first five rows, the last one. That is what .iloc does — i for integer position. It works like indexing a Python list, and it is 0-based, so the first row is position 0:

A single row comes back as a Series, with the column names now acting as its index. Reading down it, row 0 is an Adelie penguin seen on Torgersen Island.

To grab a range of rows, slice with a colon. .iloc[:5] means “positions 0 up to but not including 5” — the same half-open slicing Python lists use, so it returns exactly five rows:

This is exactly what df.head() gave you last chapter; .iloc makes the positions explicit, so you can also ask for df.iloc[10:13] or the last row with df.iloc[-1]. Position-based selection never cares what the row labels are — only where the rows sit.

Picking rows by label with .loc

.loc selects by label — the value in the row index, and the column name. For the penguins table the index is the default 0, 1, 2, …, so a label often looks like a position, but it is a different idea: .loc asks “the row labelled 0”, not “the row in position 0”. You can name the column in the same call:

Two things to notice. First, df.loc[0, "species"] reaches a single cell — the species of the row labelled 0, which is Adelie. Second, df.loc[0:4, ...] returns five rows, labels 0 through 4 — .loc slices are inclusive of the end label. That is the one gotcha worth memorising:

  • .iloc[0:4] → positions 0, 1, 2, 3 — four rows (end excluded).
  • .loc[0:4] → labels 0, 1, 2, 3, 4 — five rows (end included).

Use .iloc when you think in positions (“the first ten rows”) and .loc when you think in labels (“rows 0 through 4, and only these two columns”). Mixing them up is the most common early pandas bug, and naming which one you mean fixes it.

Filtering rows with a boolean mask

The selection you’ll reach for most is the filter: keep only the rows that meet a condition. It happens in two steps. First, compare a column to a value — this runs the test on every row at once and produces a boolean mask, a Series of True/False:

The mask has one entry per row: True where the penguin weighs more than 4000 g, False otherwise. Now pass that mask back into the DataFrame in square brackets, and pandas keeps only the True rows:

Out of 344 penguins, 172 weigh more than 4000 g. You can read df[df["body_mass_g"] > 4000] as a single sentence: “from df, take the rows where body_mass_g is over 4000.” Filtering with == works the same way for text — this is how you isolate one species:

That filter keeps 124 Gentoo penguins, and selecting the body_mass_g column of that subset and averaging it gives a mean of about 5076.02 g — Gentoos are the heavyweights of this dataset, well above the 4201.75 g average across all three species. Combining a mask (to choose rows) with a column selection (to choose what to measure) is the core loop of exploratory data analysis: narrow the rows, then read a number off them.

Key Takeaways

Key Takeaways

  1. df['col'] selects one column as a Series; df[['a', 'b']] (a list of names) selects several columns as a DataFrame.
  2. .iloc selects by integer position (0-based, end-exclusive slices, like Python lists); .loc selects by label (and its slices are inclusive of the end label).
  3. A boolean mask — df['col'] > value — is a Series of True/False; df[mask] keeps only the rows that are True.
  4. Narrow the rows with a mask, then read a column off the subset — for penguins, df[df['species'] == 'Gentoo'] keeps 124 rows.

Exercise 1 — Guided (≈4 min) 🟢

Build a DataFrame from a dictionary, take its first n rows with .head(n), and return how many rows that is as an int. Replace ____, then press Run.

If you’re stuck Build the table with pd.DataFrame(data), take the first rows with .head(n), then read the row count from .shape[0] (the first item of the (rows, columns) tuple). Wrap it in int(...) so the return type is a plain integer.
Show solution 
import pandas as pd

def first_n(data, n):
    return int(pd.DataFrame(data).head(n).shape[0])
pd.DataFrame(data) builds the table, .head(n) keeps the first n rows, and .shape[0] is the number of rows that remain. Asking for head(0) keeps no rows, so the count is 0.

Exercise 2 — Implement (≈8 min) 🟡

Write count_above(data, col, threshold): build a DataFrame, then return how many rows have col greater than threshold. This is the boolean-mask pattern — compare a column, then count the True values. Complete the marked line.

If you’re stuck pd.DataFrame(data)[col] is the column as a Series. Comparing it with > threshold gives a boolean mask (True/False per row). Calling .sum() on a boolean Series counts the True values, because True counts as 1; wrap that in int(...).
Show solution 
import pandas as pd

def count_above(data, col, threshold):
    return int((pd.DataFrame(data)[col] > threshold).sum())
(pd.DataFrame(data)[col] > threshold) is the boolean mask; .sum() adds up its True values (each worth 1). With threshold 2 the values 3 and 4 pass, so the count is 2; with threshold 9 none pass, so it is 0.

Where this fits / what’s next

You can now reach into a table three ways: pick columns by name, pick rows by position (.iloc) or label (.loc), and filter rows with a boolean mask. Those moves combine into nearly every question you’ll ask of data. Next we go from selecting what’s already there to cleaning it — handling the missing values that .describe() quietly skipped, so your selections and summaries rest on solid ground.
