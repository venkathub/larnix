Train your first model — in your browser

Beginner Browser Stable

The hook

You are about to train a real machine-learning model — in this browser tab, with nothing installed. No account, no download, no setup. Scroll to the first code block, press Run, and in a few seconds a model will sort flowers it has never seen before, and tell you how often it gets them right.

The flowers are irises. Picture three species laid out on a table. For each flower someone measured four things: the length and width of its petals and of its sepals (the green leaf-like parts under the petals). Given those four numbers, can a computer learn to name the species? That is the whole task — and by the end of this page you will have built the thing that does it.

By the end of this chapter you’ll be able to train a classifier, call it on new data, and read its accuracy.

The idea, in plain language

Start with the word that does the heavy lifting: model.

A model is a function that you learn from examples instead of writing by hand. You do not sit down and code the rule “if the petals are long and narrow, it’s species B.” Instead you show the computer many flowers whose species you already know, and it works out the rule for you.

Three words describe the whole cycle:

  • Training (also called fitting): you hand the model a pile of examples where the answer is known — here, flowers labelled with their species — and it adjusts itself until its guesses match those known answers reasonably well.
  • Predicting: you give the trained model a new flower it has not seen, and it outputs a species.
  • Accuracy: you check how often it is right. If you test it on 40 flowers and it names 38 correctly, that is 38/40 = 0.95, or 95% accuracy.

One habit matters from day one: you test on flowers the model did not train on. A model can score well on examples it has already memorised and still fail on new ones, the way reciting answers you’ve seen is not the same as understanding. So we split the data into a training set (to learn from) and a test set (to grade on).

Key Takeaways

Key Takeaways

  1. A model is a function learned from labelled examples, not a rule you write by hand.
  2. Training (fitting) adjusts the model on known answers; predicting runs it on new inputs.
  3. Accuracy is the fraction of predictions that match the truth — always measured on data the model did not train on.
  4. You did all of this in the browser, at zero cost and zero install.

Exercise 2 — Implement (≈10 min) 🟡

Write accuracy(y_true, y_pred) — the fraction of predictions that match the truth, as a number between 0 and 1. Complete the marked line.

Show solution 
def accuracy(y_true, y_pred):
    correct = sum(1 for a, b in zip(y_true, y_pred) if a == b)
    return correct / len(y_true)
a == b is True when a prediction matches the truth; summing those Trues counts the matches, and dividing by the number of labels gives the fraction. This is what accuracy_score computed for you in Step 5.

Open-ended, rubric-graded (not auto-checked). Starting from the worked example, try to beat 95% accuracy on the test set. Ideas: raise max_iter, or swap LogisticRegression for another classifier such as from sklearn.neighbors import KNeighborsClassifier.

Rubric — a good solution:

  • Trains on X_train/y_train only and reports accuracy on the held-out X_test/y_test (no testing on training data).
  • States the change you made and the accuracy you reached.
  • Reaches at least the worked example’s baseline, ideally higher.
Show one possible solution 
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

knn = KNeighborsClassifier(n_neighbors=5)
knn.fit(X_train, y_train)
print(f"k-NN accuracy: {accuracy_score(y_test, knn.predict(X_test)):.2%}")
A nearest-neighbour classifier often does well on Iris. Other approaches are fine too — the point is to change one thing, measure honestly on the test set, and compare.
