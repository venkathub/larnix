A NumPy preview

Beginner Browser Stable

The hook

Suppose you want to double every step count in a week, or count how many days you hit your goal. With a plain Python list you reach for a loop — write it, track an index or a running total, append to a new list. NumPy lets you say what you want to the whole column at once: steps * 2 doubles every value, and steps >= 8000 tests every day in one go. No loop, no bookkeeping.

That “do the maths on the entire array at once” idea is called vectorized thinking, and it’s the foundation of the data and AI tools coming in M2 and M3. This chapter is your first taste.

You’ll need from before: loops and conditions (Ch2) — the very thing NumPy lets you stop writing by hand — and lists (Ch4), which is what an array is built from.

What is a NumPy array?

NumPy is Python’s numerical-computing library. Its core type is the array: an ordered container of numbers that all share one type. You import the library under the customary short name and build an array from a list:

import numpy as np

steps = np.array([7000, 8200, 9000, 6500, 8000])

A list and an array look similar, but an array is built for maths on the whole collection. That difference is the whole point of this chapter.

Element-wise operations (no loop needed)

Do arithmetic on an array and the operation applies to every element, giving back a new array of the same length:

steps * 2        # every value doubled
steps + 100      # 100 added to every value
steps >= 8000    # a True/False for every value

The last one returns a boolean array — the result of testing each element against 8000. Compare that with a list, where steps * 2 would repeat the list and steps >= 8000 would be an error. With an array, the operation reaches every element for you — this is what “vectorized” means.

Summary methods

An array carries methods that reduce all its values to one number:

steps.mean()     # the average
steps.sum()      # the total
steps.max()      # the largest value

One note for later: steps.mean() gives back a NumPy float, not a plain Python float. It prints and compares like a number, so for everyday use you won’t notice the difference.

Boolean masking

A boolean array can select elements. Put the test inside the square brackets and you get back only the elements where it was True:

steps[steps >= 8000]      # the values that met the goal
(steps >= 8000).sum()     # how many met it (True counts as 1)

steps >= 8000 builds the mask; steps[...] keeps the matching values; .sum() on the mask counts the Trues, because True behaves as 1 and False as 0. When you want a plain Python int from that count, wrap it: int((steps >= 8000).sum()).

Note🧱 For Java developers

A NumPy array is closer to a typed numeric buffer than to int[] plus a loop. In Java you’d write for (int i = 0; i < a.length; i++) b[i] = a[i] * 2;. In NumPy a * 2 expresses the same intent in one line, and the library runs the per-element work in compiled code — closer in spirit to bulk or SIMD-style math than to element-by-element iteration. The boolean mask a[a >= 8000] replaces the filter-into-a-new-list loop entirely.

Let’s build it (worked example)

A week of step counts, handled without a single loop. Press Run.

steps * 2 reaches every value; steps >= 8000 tests every day at once and gives back [False True True False True]. No index, no loop.

The mask steps >= goal does double duty: .sum() counts the Trues (three days) and steps[mask] keeps the matching values. Wrapping the count in int(...) gives a plain Python integer, handy when exact equality matters.

Key Takeaways

Key Takeaways

  1. A NumPy array (np.array([...])) holds numbers of one type and is built for maths on the whole collection.
  2. Element-wise operations like arr * 2 and arr >= 8000 apply to every element at once — no Python loop.
  3. Summary methods arr.mean(), arr.sum(), arr.max() reduce the whole array to a single value.
  4. A boolean mask selects (arr[arr >= 8000]) and counts (int((arr >= 8000).sum())) the matching elements.

import numpy as np

def array_mean(values):
    arr = np.array(values)

run_tests([

arr.mean() averages every element of the array. [8000, 9000] averages to 8500.0; three equal values average to that same value.

Exercise 2 — Implement (≈8 min) 🟡

Write count_active(steps, goal) — build an array and return, as a plain int, how many step counts are at least goal. Use a boolean mask. Complete the marked line.

If you’re stuck The mask is arr >= goal. Counting its True values is (arr >= goal).sum(), and int(...) around it returns a plain Python integer.
Show solution 
import numpy as np

def count_active(steps, goal):
    arr = np.array(steps)
    return int((arr >= goal).sum())

run_tests([
    ("two of three", count_active([7000, 8200, 9000], 8000), 2),
    ("none qualify", count_active([100, 200], 8000), 0),
    ("both qualify", count_active([8000, 9000], 8000), 2),
])
arr >= goal builds a boolean mask; .sum() counts the Trues because True acts as 1; int(...) returns a plain Python integer. With goal 8000, the first list has two qualifying days, the second none, the third two.

Exercise 3 — Stretch (optional, ≈8 min) 🔴

Open-ended, rubric-graded (not auto-checked). Use a boolean mask to select the step values that meet a goal and print them. Build an array, make the mask, and index with it.

Rubric — a good answer:

  • Builds an array with np.array(...) and a goal value.
  • Uses a boolean mask to index the array: arr[arr >= goal].
  • Prints the selected values and runs on a mix of above- and below-goal days.
Show one possible answer 
import numpy as np

def active_values(steps, goal):
    arr = np.array(steps)
    return arr[arr >= goal]

print(active_values([7000, 8200, 9000, 6500, 8000], 8000))   # [8200 9000 8000]
arr >= goal is a boolean mask, and arr[mask] keeps only the elements where the mask is True. Here three days meet the 8000 goal, so those three values print.
