Pandas Series & DataFrame

Beginner Browser Stable

The hook

Open a spreadsheet and you already understand the shape of pandas. A spreadsheet is a table: rows of records, columns with names, and a header strip along the top. Pandas gives you that same table in Python — except now you can sort, filter, average, and reshape millions of rows with a line of code instead of dragging a mouse.

Pandas has exactly two objects you need to meet first, and everything else builds on them:

  • A Series — one column. A labelled list of values, like a single column of the spreadsheet pulled out on its own.
  • A DataFrame — the whole table. A collection of named columns sitting side by side, each column being a Series.

That’s the entire mental model. In this chapter we load a real dataset — 344 penguins measured at Palmer Station, Antarctica — and learn the handful of commands that let you size up any table the moment it lands on your screen.

You’ll need from before: NumPy arrays and vectorized thinking (M3 Ch1). A pandas column is a NumPy array wearing a name tag, so that intuition carries straight over.

A Series is a labelled column

A Python list holds values by position — values[0], values[1], and so on. A Series holds values too, but it pairs each value with a label (its index). By default the labels are 0, 1, 2, …, but the point is that the label travels with the value wherever it goes.

Notice the two columns of output: the index (0, 1, 2, 3) on the left and the values on the right, plus a dtype (here int64) telling you what kind of data this column holds. A Series is one-dimensional — a single line of data with labels.

Because a Series is built on NumPy, whole-column arithmetic and summaries come for free. There is no loop to write:

masses.mean() collapses the column to one number; masses / 1000 returns a new Series, dividing every value at once while keeping the same labels. That is the NumPy “operate on the whole array” idea, now with an index attached.

A DataFrame is a table of Series

Stack several labelled columns side by side, sharing one index, and you have a DataFrame — pandas’ name for a table. Each column is a Series; the column names let you ask for the one you want.

You can build a small DataFrame straight from a dictionary, where each key becomes a column name and each value becomes that column’s data:

Two named columns, one shared index down the left. Real datasets, though, rarely arrive as hand-typed dictionaries — they arrive as files. So let’s load one.

Loading a CSV and sizing it up

We’ll use the Palmer Penguins dataset: body measurements for 344 penguins of three species. The chapter ships the CSV alongside it, and the load_csv helper reads it for you — the same call works in your browser and in our test runner.

The first thing to check on any new table is its shape. df.shape returns a tuple (rows, columns) — here (344, 8). That single line tells you the dataset has 344 records described by 8 columns, before you’ve looked at a single value.

Next, peek at the actual rows with .head(), which returns the first five by default:

Now you can see what each row is: a penguin, with its species, the island it was seen on, four body measurements, its sex, and the study year. To get the column names on their own — handy when a table is too wide to read at a glance — ask for .columns:

That returns the eight names: species, island, bill_length_mm, bill_depth_mm, flipper_length_mm, body_mass_g, sex, year. The _mm and _g suffixes are a gift from whoever prepared the data — the units are baked into the names.

Summarising every numeric column at once

.describe() is the fastest way to understand the numbers in a table. For every numeric column it reports the count, mean, standard deviation, min, max, and the quartiles — a one-line statistical X-ray:

Read down the body_mass_g column of that summary and you’ll find a mean of about 4201.75 grams — these penguins average a little over four kilograms. The flipper_length_mm mean lands near 200.92 mm. Note the count row: it can be lower than 344 for some columns, because a few penguins have missing measurements — .describe() quietly skips them, which is a detail worth noticing early.

Pulling out one column

To work with a single column, index the DataFrame with its name in square brackets. What comes back is a Series — that same labelled column from the start of the chapter, now sourced from real data:

df["body_mass_g"] is a Series, so every Series tool applies: .mean() gives 4201.75, matching what .describe() reported. Selecting a column and summarising it is the bread-and-butter move of exploratory data analysis, and you’ll repeat it constantly.

One more useful summary for a text column is .value_counts(), which tallies how often each value appears:

That shows Adelie 152, Gentoo 124, Chinstrap 68 — Adelie is the most common species in the dataset. With .shape, .head, .columns, .describe, column selection, and .value_counts, you can walk up to almost any table and know what it holds within a minute.

Key Takeaways

Key Takeaways

  1. A Series is a labelled 1-D column: values paired with an index, built on NumPy so whole-column maths needs no loop.
  2. A DataFrame is a 2-D table — several named columns (each a Series) sharing one index.
  3. df.shape returns (rows, columns); for penguins it is (344, 8). .head(), .columns, and .describe() size up a table fast.
  4. Selecting a column with df['col'] returns a Series, so every Series summary like .mean() applies to it directly.

Exercise 1 — Guided (≈4 min) 🟢

Build a Series from a list of numbers and return its total as an int. Complete series_total by replacing ____, then press Run.

If you’re stuck Wrap the list in pd.Series(...), call .sum() on it, and pass the result to int(...) so the return type is a plain integer.
Show solution 
import pandas as pd

def series_total(values):
    return int(pd.Series(values).sum())
pd.Series(values) turns the list into a labelled column; .sum() adds every value, and int(...) returns it as a plain Python integer.

Exercise 2 — Implement (≈8 min) 🟡

Build a DataFrame from a dictionary, select one column by name, and return the rounded mean of that column. Complete the marked line.

If you’re stuck pd.DataFrame(data) builds the table; [col] selects one column as a Series; .mean() averages it. Wrap that in float(...), then round(..., 2) for two decimal places.
Show solution 
import pandas as pd

def column_mean(data, col):
    return round(float(pd.DataFrame(data)[col].mean()), 2)
pd.DataFrame(data)[col] is the column as a Series; .mean() averages it, float(...) makes the type predictable, and round(..., 2) trims it to two decimal places.

from lib.data import load_csv
import pandas as pd

df = load_csv("penguins")
print(df.describe())
