Linear algebra in NumPy

Intermediate Browser Stable

The hook

Chapters 2 through 5 were pictures and arrows: a vector as a point, a matrix as a grid that moves points around, a dot product as a way to measure similarity. Those pictures built the intuition. But a model never draws an arrow — when it needs a dot product or a matrix multiply, it calls NumPy, millions of times a second.

This chapter does the same operations you already understand, but in real code, in one tool. Nothing new conceptually — you are translating the pictures into the lines a model actually runs.

You’ll need from before: dot products and similarity (M2 Ch5), and the NumPy array basics (M1 Ch12). That’s the toolkit we build on here.

Vectors and matrices are arrays

In NumPy, a vector is a 1-D array and a matrix is a 2-D array (a list of rows). You build both with np.array:

The .shape attribute is the single most useful thing to read off an array. It is a tuple giving the size along each dimension:

  • v.shape is (3,) — one dimension, length 3. (The trailing comma is Python’s way of writing a one-element tuple; it is not a typo.)
  • M.shape is (2, 3) — two dimensions: 2 rows then 3 columns. The convention is always (rows, columns).

When code breaks in real machine learning, it is almost always a shape that doesn’t line up. Reading .shape is your first debugging move, every time.

Element-wise is not the dot product

Multiply two arrays with * and NumPy multiplies them position by position — this is element-wise, and it gives back an array the same shape as the inputs. That is not the dot product. The dot product multiplies position by position and then adds the results into a single number.

All three middle lines agree: 1·4 + 2·5 + 3·6 = 4 + 10 + 18 = 32. The @ operator is the one you’ll see most in real code — it is Python’s dedicated “matrix multiply” operator, and for two vectors it computes exactly the dot product from Ch5. np.dot(a, b) does the same job; @ reads more cleanly when expressions get longer.

A matrix times a vector: apply a transformation

Back in Ch3, a matrix was a machine that takes a point and moves it. In NumPy that machine is one @. Multiplying a matrix M by a vector v applies the transformation M describes to the point v:

Read it row by row: the first output number is the first row of M dotted with v (2·4 + 0·5 = 8), and the second is the second row dotted with v (0·4 + 3·5 = 15). A matrix–vector product is one dot product per row. That is the whole idea: a layer of a neural network takes its input vector and does exactly this — a matrix times a vector — to produce the next vector.

The shape rule is worth saying out loud: a (2, 2) matrix times a (2,) vector gives a (2,) vector. In general, multiplying a (rows, k) matrix by a length-k vector gives a length-rows vector. The inner sizes must match.

A matrix times a matrix

Stacking transformations means multiplying matrices together, again with @. Each entry of the result is one row of the left matrix dotted with one column of the right matrix:

The top-left entry is row 0 of A dotted with column 0 of B: 1·5 + 2·7 = 19. Work the other three the same way and you get the full grid. For matrix–matrix multiplication the rule is (m, k) @ (k, n) gives (m, n): the two inner numbers must match and they cancel, leaving the outer two as the result shape.

Transpose: flipping rows and columns

The transpose, written .T, flips a matrix over its diagonal: rows become columns and columns become rows. The element at row i, column j ends up at row j, column i.

Transpose shows up constantly: to make shapes line up for an @, or to read the same data the other way around. One handy fact you’ll meet again: (A @ B).T equals B.T @ A.T — transposing a product reverses the order. You’ll verify that yourself in the stretch exercise.

The identity matrix

One special matrix does nothing: the identity, ones on the diagonal and zeros elsewhere. Build it with np.eye(n). Multiplying any vector or matrix by the identity returns it unchanged — it is the matrix equivalent of multiplying a number by 1.

The identity is the reference point for talking about inverses and “undoing” a transformation later. For now, it is a clean check that your @ is doing what you expect: an identity should always hand back exactly what it received.

Key Takeaways

Key Takeaways

  1. A vector is a 1-D NumPy array and a matrix is a 2-D array; read .shape (as (rows, columns)) to know what you’re holding.
  2. * multiplies element-wise; @ (and np.dot) computes the dot product for vectors and matrix multiplication for 2-D arrays.
  3. A matrix times a vector applies a transformation — one dot product per row; the inner sizes must match.
  4. .T transposes (flips rows and columns), and np.eye(n) is the identity matrix that leaves things unchanged.

Exercise 1 — Guided (≈4 min) 🟢

Complete dot(a, b) so it returns the dot product of two vectors as a plain int. Use @ on NumPy arrays, then wrap the result in int(...). Replace ____, then press Run.

If you’re stuck Turn each list into an array with np.array(...), combine them with @ to get the sum of products, then wrap the whole thing in int(...) so the result is a plain Python integer rather than a NumPy number.
Show solution 
import numpy as np

def dot(a, b):
    return int(np.array(a) @ np.array(b))
np.array(a) @ np.array(b) multiplies the vectors position by position and adds the results: 1·4 + 2·5 + 3·6 = 32. The orthogonal pair [1, 0] and [0, 1] share no direction, so their dot product is 0. int(...) returns a plain Python integer.

Exercise 2 — Implement (≈8 min) 🟡

Write matvec(M, v) that applies matrix M to vector v and returns the result as a plain Python list. Convert both inputs with np.array, multiply with @, and turn the array back into a list with .tolist(). Complete the marked line.

If you’re stuck The pattern mirrors Exercise 1: np.array(M) @ np.array(v) does the matrix–vector multiply (one dot product per row of M). The result is a NumPy array, so call .tolist() on it to hand back an ordinary Python list.
Show solution 
import numpy as np

def matvec(M, v):
    return (np.array(M) @ np.array(v)).tolist()
Each output entry is one row of M dotted with v. With the identity matrix every row picks out one element of v unchanged, so [5, 7] comes straight back. The scaling matrix [[2, 0], [0, 3]] doubles the first entry and triples the second: [8, 15]. .tolist() converts the NumPy array to a plain list.

Exercise 3 — Stretch (optional, ≈8 min) 🔴

Open-ended, rubric-graded (not auto-checked). Build two 2×2 matrices A and B as NumPy arrays, compute A @ B, and verify the identity (A @ B).T == B.T @ A.T — transposing a product reverses the order. Print both sides and compare them.

Rubric — a good answer:

  • Builds two 2×2 arrays with np.array and computes A @ B with @.
  • Prints (A @ B).T and B.T @ A.T so the two sides can be read side by side.
  • Confirms they are equal with np.array_equal(...) and prints the True result.
If you’re stuck Pick any numbers for A and B. Compute the left side as (A @ B).T and the right side as B.T @ A.T. To compare two whole arrays for equality, use np.array_equal(left, right) — comparing with plain == gives an array of True/False rather than a single answer.
Show one possible answer 
import numpy as np

A = np.array([[1, 2],
              [3, 4]])
B = np.array([[5, 6],
              [7, 8]])

left = (A @ B).T
right = B.T @ A.T

print("(A @ B).T =")
print(left)
print("B.T @ A.T =")
print(right)
print("equal?", np.array_equal(left, right))
Both sides print the same grid, and np.array_equal returns True: transposing a product flips the order of the factors. This identity is why you’ll often see .T rearranged across an @ when lining up shapes in later chapters.
